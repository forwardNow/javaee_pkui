issue

有一些问题（bug）未能（完全）解决，在此以做记录。

#1 在IE8上，有些 require(“依赖”)虽然下载了，但未执行完毕

问题描述：

/* 参考：desktop/js/config/globalSetting.js */

define( function( require ) {
    var
        $ = require( "jquery" ),
        PKUI = window.PKUI
    ;

    // 日期控件
    require( "datepicker" );

    // 日期控件 -- 设置默认的日期格式
    $.fn.datepicker.dates[ "zh-CN" ].format = "yyyy-mm-dd";
} );

执行 require( "datepicker" ); 后，在IE8上，有时下载完“datepicker”后 但未执行完毕，导致后面的语句执行报错。

解决方案：

由于，此异常发生不是很频繁，且通过刷新暂时可以解决问题。

所以，对window的error事件注册事件处理函数，判断是否是上述异常，如果是 则刷新。

但是，未能真正解决问题，目前判断应该是 seajs v2.3.0 的bug，

很遗憾 seajs 的作者已经不再维护它，且搜索相关的issue也未能找到解决方案。

因此，只能再研究完seajs的源码后，才可以尝试去解决该异常。
