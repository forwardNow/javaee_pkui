弹出层

弹出层

1. 介绍

desktop 引入了三种弹出层组件：

  • art-dialog：用于（app）快捷方式，作为app的弹出层容器使用
  • layer：用于 alert（警告框）、confirm（确认框）、message（消息框）、页面层
  • drawer：PKUI组件，用于 app 内的弹出层

2. 使用

2.1 art-dialog

art-dialog v6.0.4.x 已被完全修改为只适用于 desktop 的“快捷方式app应用”，

因此，关于 art-dialog 的使用 将会纳入快捷方式的使用说明里，

所以，在此不做赘述。

2.2 layer

官网：http://www.layui.com/layer/

由于原生的弹出层组件不可设置其样式，因此引入 layer（v3.0.1） 以丰富：

  • alert（警告）
  • confirm（确认/询问）
  • message（消息）
  • 页面层（详情页/编辑页）

layer组件属于预加载模块，可直接使用（window.layer.alert()）。

2.2.1 open

layer.open(options)

  • 原始核心方法。不管是使用哪种方式创建层，都是通过layer.open()
  • 参数：参考“常用参数”
  • 返回值：返回一个当前层索引。

2.2.2 参数（options）

  • type
    • 基本层类型
    • 类型：Number，默认：0
      • 0（信息框，默认）
      • 1（页面层）
      • 2（iframe层）
      • 3（加载层）
      • 4（tips层）
  • title
    • 说明：标题
    • 类型：String/Array/Boolean，默认：'信息'
      • String：改变标题文本，如 title :'我是标题'
      • Array：自定义标题区域样式，如 title: ['文本', 'font-size:18px;color:#333;']
      • Boolean：如果你不想显示标题栏，设置title: false
  • content
    • 内容
    • 类型：String/DOM/Array，默认：''
      • String：普通的html内容
  • area
    • 作用：宽高
    • 类型：String/Array，默认：'auto'
      • String：在默认状态下，layer是宽高都自适应的，当你只想定义宽度时，你可以area: '500px'，高度仍然是自适应的
      • Array：当你宽高都要定义时，你可以area: ['500px', '300px']
  • offset
    • 作用：坐标
    • 类型：String/Array，默认：垂直水平居中
      • String：但如果你不想垂直水平居中，你还可以offset: '100px' ，只定义top坐标，水平保持居中
      • Array：offset: ['100px', '50px'] 同时定义top、left坐标
  • icon
    • 作用：图标。
    • 类型：Number，默认：-1（不显示）
      • 0：橙色的 警告
      • 1：绿色的 成功
      • 2：红色的 错误
      • 3：黄色的 疑问
      • 4：深灰的 锁
      • 5：红色的 失望表情
      • 6：绿色的 高兴表情
  • btn
    • 作用：按钮。信息框模式时，btn默认是一个确认按钮，其它层类型则默认不显示。
    • 类型：String/Array，默认：'确认'
      • String：当您只想自定义一个按钮时，你可以btn: '我知道了'
      • Array：当你要定义两个及以上按钮时，你可以btn: ['确认', '取消']
        • 按钮1的回调为 yes:function(){}
        • 按钮2的回调为btn2:function(){}
        • 按钮3的回调为btn3:function(){}
        • ...
  • btnAlign
    • 作用：按钮排列
    • 类型：String，默认："r"
      • btnAlign: 'l' 按钮左对齐
      • btnAlign: 'c' 按钮居中对齐
      • btnAlign: 'r' 按钮右对齐
  • closeBtn
    • 作用：右上角的 关闭按钮
    • 类型：Number，默认：1
      • 0：不显示
      • 1：在标题内
      • 2：在标题外
  • shade
    • 作用：遮罩层
    • 类型：Array/Boolean，默认：[0.3, "#000"]（0.3透明度的黑色背景）
      • 如果你不想显示遮罩，可以shade: 0
  • shadeClose
    • 作用：是否点击遮罩关闭
    • 类型：Boolean，默认：false
  • time
    • 作用：设置自动关闭所需毫秒数
    • 类型：Number，默认：0（不自动关闭）
  • anim
    • 作用：弹出动画（出场动画全部采用CSS3，ie6-9无效果）
    • 类型：Number，默认：0
      • anim: 0 平滑放大。默认 anim: 1 从上掉落 anim: 2 从最底部往上滑入 anim: 3 从左滑入 anim: 4 从左翻滚 anim: 5 渐显 anim: 6 抖动
  • maxmin
    • 作用：最大最小化。
    • 类型：Boolean，默认：false
  • fixed
    • 作用：固定。即鼠标滚动时，层是否固定在可视区域
    • 类型：Boolean，默认：true
  • resize
    • 作用：是否允许拉伸（拖动弹层右下角来拉伸尺寸）
    • 类型：Boolean，默认：true
  • scrollbar
    • 作用：是否允许浏览器出现滚动条。默认允许浏览器滚动，如果设定scrollbar: false，则屏蔽
    • 类型：Boolean，默认：true
  • maxWidth
    • 作用：最大宽度。只有当area: 'auto'时，maxWidth的设定才有效。
    • 类型：Number，默认：360
  • maxHeight
    • 作用：最大高度。只有当高度自适应时，maxHeight的设定才有效。
    • 类型：Number，默认：无
  • zIndex
    • 作用：层叠顺序。一般用于解决和其它组件的层叠冲突。
    • 类型：，默认：19891014
  • move
    • 作用：触发拖动的元素。默认是触发标题区域拖拽，如果你想单独定义，指向元素的选择器或者DOM即可。
    • 类型：String/DOM/Boolean，默认：'.layui-layer-title'。false，禁止拖拽。
  • moveOut
    • 作用：是否允许拖拽到窗口外。默认只能在窗口内拖拽，如果你想让拖到窗外，那么设定moveOut: true即可
    • 类型：Boolean，默认：false
  • moveEnd
    • 作用：拖动完毕后的回调方法
    • 类型：Function，默认：null
      • 默认不会触发moveEnd，如果你需要，设定moveEnd: function(layero){}即可，其中layero为当前层的DOM对象
  • success
    • 作用：层弹出后的成功回调方法。当你需要在层创建完毕时即执行一些语句，可以通过该回调
    • 类型：Function，默认：null
      • function(layero, index){ }，layero 当前层，index当前层索引。
  • yes
    • 作用：确定按钮回调方法。第一个按钮的回调。如果设定了yes回调，需进行手工关闭
    • 类型：Function，默认：null
      • function(index, layero){ //do something layer.close(index); //如果设定了yes回调，需进行手工关闭 } });
  • cancel
    • 作用：右上角关闭按钮触发的回调。
    • 类型：Function，默认：null。默认会自动触发关闭。如果不想关闭，return false即可。
      • function(index, layero){ if(confirm('确定要关闭么')){ //只有当点击confirm框的确定时，该层才会关闭 layer.close(index) } return false; }
  • end
    • 层销毁后触发的回调。无论是确认还是取消，只要层被销毁了，end都会执行，不携带任何参数。
    • 类型：Function，默认：null
  • full/min/restore
    • 分别代表最大化、最小化、还原 后触发的回调
    • 类型：Function，默认：null。携带一个参数，即当前层DOM

2.2.3 alert

在 pkui.js 文件中，全局函数alert()被覆写：

/**
 * 修改浏览器的原生方法（如 alert ）
 */
_modifyBrowserMethod: function () {
    window.__alert = window.alert;
    window.alert = window.layer.alert;
}

示例（参数请参考“2.2.2 参数（options）”）：

window.alert( 
    // 内容
    "保存成功！", 
    // 参数
    {
        // 标题
        title: "信息",
        // 图标：0 - 警告；1 - 成功；2 - 失败；3 - 疑问；
        icon: 1,
        // 按钮：数组元素个数绝对多少个按钮，元素值为按钮文字
        btn: [ "确定" /*, "取消"*/ ]

    } 
);

2.2.4 confirm

参数请参考“2.2.2 参数（options）”

layer.confirm( "是否删除",
    {
        title: "操作确认",
        btn: [ "删除" , "取消" ] //按钮
    },
    // 确认回调
    function( index ){
        // 删除操作...
        // 手动关闭当前确认框
        layer.close( index );
    },
    // 取消回调
    function(){

    }
);

2.2.5 message

参数请参考“2.2.2 参数（options）”

layer.msg( '网络错误！', { icon: 0 } );
layer.msg( '删除成功！', { icon: 1 } );
layer.msg( '删除失败！', { icon: 2 } );

2.2.6 页面层

通常用于弹出页面，显示一个独立的页面。

seajs.use( [ "jquery", "template" ], function ( $, Template ) {
    var 
        view = "__CTX__/static/desktop/tpl/system/dept/detail.html",
        model = "__CTX__/admin/sysDeptModel?deptId=420000000000"
    ;    

    Template.getModelAndView( view, model, function ( htmlString ) {
        window.layer.open( {
            title: "查看部门",
            type: 1,
            skin: 'layui-layer-rim', //加上边框
            area: ['800px', '420px'], //宽高
            content: htmlString,
            success: function ( layero, index ) {
                // 暂存存储绑定弹出层索引，
                // 以便 通过 window.layer.close( index ) 关闭
                layero.find("form table input[name='layerIndex']").val( index );
            }
        } );
    } );

} );

2.3 drawer

此PKUI组件，应用场景不多，暂时在app窗口内使用。

查看源文件：https://github.com/forwardNow/pkui/blob/master/src/component/common/drawer/1.0.0/drawer.js

2.3.1 说明

参数：

  • container
    • 作用：指定弹出层的容器，或者是指定弹出层完全覆盖哪个元素。
    • 值：String（CSS选择） | jQuery。如果有多个，则从 target节点 往上查找最近的祖先元素
  • referenceElementSelector（v1.0.11）
    • 作用：如果 container参数 指定选择器不唯一，则会以referenceElementSelector 指定的元素为参考点找最近的符合container参数值的祖先元素
  • url
    • 作用：指定模板的URL
  • model
    • 作用：指定模型URL；不指定，则直接载入url参数指定的模板（比如JSP文件）

方法：

  • open
    • 作用：弹出抽屉层
  • hide
    • 作用：隐藏抽屉层
  • destroy
    • 作用：销毁抽屉层。点击drawer弹出层的关闭按钮后会自动调用此方法。

2.3.2 结合bootgrid使用

一般的用法，结合 bootgrid 使用，以滑出抽屉的形式产生一个弹出层。

查看源文件：https://github.com/forwardNow/pkui/blob/master/app/desktop/tpl/system/dept/list.html

<table id="sysDeptDatagrid"
       data-pkui-component="datagrid"
    ...
    <th ... data-formatter="sysDeptOpe">操作</th>
    ...    
</table>

    function sysDeptOpe(column, row) {
        return '<a href="javascript:void(0)" '
        +           ' id=viewSysDept_' + row.deptId
        +           ' data-pkui-component="drawer"'
        +           ' data-pkui-component-options=\'{'

        +               '"container": ".da-win-main"}\'><i class="fa fa-file-o"></i> 查看</a>　'
        ;
    }
</script>

2.3.3 主动弹出抽屉层

有些情况，不是结合bootgrid使用的，这就得主动弹出抽屉层。

为满足这种应用场景，对 drawer.js 进行了一些扩展，版本为：PKUIv1.0.11。

打开抽屉：window.PKUI.component.drawer( "open", options );

seajs.use( "drawer", function () {
    var options = {
        "url":"__CTX__/static/desktop/tpl/system/dept/detail.html",
        "model":"http://www.20ui.cn/admin/sysDeptModel?deptId=420000000000",

        // 由于 container 指定的容器选择器不唯一，
        // 但是 容器里的某个元素是唯一的，如 "#sysDeptDatagrid"
        // 因此 $( "#sysDeptDatagrid" ).closest( ".da-win-main" ) 可以确认唯一的容器
        "referenceElementSelector": "#sysDeptDatagrid",
        "container": ".da-win-main"
    }

    window.PKUI.component.drawer( "open", options );
} );
  • 你的版本得是 "1.0.11"以上的，查看版本window.PKUI.version
  • 保证 “drawer”已经载入
    • seajs.use( "drawer", function () { window.PKUI.component.drawer( "open", options ); } );
  • 指定“drawer”覆盖的容器，也就是说“drawer”在哪个容器里打开
    • 如果容器的选择器是唯一的，则只需指定参数container: "#container-xx"
    • 如果容器的选取不唯一，则需指定容器内的可以唯一确认的参考点 referenceElementSelector

查看源文件：https://github.com/forwardNow/pkui/blob/master/app/desktop/tpl/system/dept/list.html。

<table id="sysDeptDatagrid"
       data-pkui-component="datagrid"
    ...
    <th ... data-formatter="sysDeptOpe_2">操作</th>
    ...    
</table>
<script>
/**
 * 操作，手动打开 抽屉层
 */
function sysDeptOpe_2(column, row) {
    return '<a href="javascript:void(0)" '
    +           ' onclick="sysDeptOpe_2.viewDetail(this)"'
    +           ' data-options=\'{'
    +               '"url":"__CTX__/static/desktop/tpl/system/dept/detail.html",'
    +               '"model":"{% system.dept.list.sysDeptModel %}?deptId=' + row.deptId + '",'

    +               '"referenceElementSelector": "#sysDeptDatagrid",'
    +               '"container": ".da-win-main"}\'><i class="fa fa-file-o"></i> 查看</a>　'
    ;
}
sysDeptOpe_2.viewDetail = function ( target ) {
    var
        $ = jQuery,
        $target = $( target ),
        options = $target.data( "options" )
    ;
    layer.msg( '正在验证您的权限...', {
        icon: 1,
        time: 2000 // 2秒关闭（如果不配置，默认是3秒）
    }, function () {

        seajs.use( "drawer", function () {
            window.PKUI.component.drawer( "open", options );
        } );

    } );
};
</script>
