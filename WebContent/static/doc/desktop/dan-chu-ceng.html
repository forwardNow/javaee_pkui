弹出层

1. 介绍

desktop 引入了三种弹出层组件：

  • art-dialog：用于（app）快捷方式，作为app的弹出层容器使用
  • layer：用于 alert（警告框）、confirm（确认框）、message（消息框）、页面层
  • drawer：PKUI组件，用于 app 内的弹出层

2.3 drawer

此PKUI组件，应用场景不多，暂时在app窗口内使用。

查看源文件：https://github.com/forwardNow/pkui/blob/master/src/component/common/drawer/1.0.0/drawer.js

2.3.1 说明

参数：

  • container
    • 作用：指定弹出层的容器，或者是指定弹出层完全覆盖哪个元素。
    • 值：String，CSS选择器。如果有多个，则从 target节点 往上查找最近的祖先元素
  • url
    • 作用：指定模板的URL
  • model
    • 作用：指定模型URL；不指定时，则直接载入url参数指定的模板

方法：

  • open
    • 作用：弹出抽屉层
  • hide
    • 作用：隐藏抽屉层
  • destroy
    • 作用：销毁抽屉层。点击drawer弹出层的关闭按钮后会自动调用此方法。

注意：

  • 由于 drawer 是PKUI组件，得有宿主元素（目标元素）

2.3.2 结合bootgrid使用

一般的用法，结合 bootgrid 使用，以滑出抽屉的形式产生一个弹出层。

查看源文件：https://github.com/forwardNow/pkui/blob/master/app/desktop/tpl/system/dept/list.html

<table id="sysDeptDatagrid"
       data-pkui-component="datagrid"
    ...
    <th ... data-formatter="sysDeptOpe">操作</th>
    ...    
</table>
<script>
    function sysDeptOpe(column, row) {
        return '<a href="javascript:void(0)" '
        +           ' id=viewSysDept_' + row.deptId
        +           ' data-pkui-component="drawer"'
        +           ' data-pkui-component-options=\'{'
        +               '"url":"__CTX__/static/desktop/tpl/system/dept/detail.html",'
        +               '"model":"{% system.dept.list.sysDeptModel %}?deptId=' + row.deptId + '",'
        +               '"container": ".da-win-main"}\'><i class="fa fa-file-o"></i> 查看</a>　'
        ;
    }
</script>
