应用

1. 介绍

菜单表（SYS_MENU）中每一条记录对应一个【菜单项】，“菜单项” 有

  • 主键（menuId）
  • 图标（icon）
  • 名称（menuName）
  • 链接（menuUrl）
  • ...

在 “desktop” 中，“菜单项”被定义为【app应用】，“app应用”通常以点击【app快捷方式】图标的方式打开：

  • 打开后显示“app应用”的窗口由侧边栏、内容区域组成
  • 侧边栏 显示“app应用”对应菜单项，当“app应用”对应菜单项为：
    • 目录时，则显示 “app应用”对应菜单项 的所有子菜单
    • 链接时，则折叠侧边栏，并载入链接的内容。
  • 内容区域 用于载入菜单项URL对应的内容

注：当“app应用”的左侧菜单栏只有一项时，左侧菜单栏会自动隐藏。

2. 使用

  • 点击“app快捷方式”图标 来打开相应的“app应用”
  • 搜索“app应用” 来打开
  • 调用“API” 来打开

2.1 “app快捷方式”

“app快捷方式”的权限管理，限制用户可以使用的“app应用”；

“app快捷方式”的配置，预定义要显示的“app应用”以及排列的次序。

2.1.1 权限管理

“app快捷方式”本质上还是“菜单项”，而“菜单项”的获取是经过权限过滤的，

所以“app快捷方式”的权限管理 也就是 “菜单项”的权限管理。

2.3 打开“app应用”的API

当所有脚本、样式载入完毕后，可以通过以下方式来打开一个“app应用”，

也就是说，你可以在已经初始化完毕的desktop的页面，打开控制台输入以下代码来打开“app应用”。

seajs.use( window.PKUI.ctxPath + "/static/desktop/js/common/app", function ( App ) {    
    var app = new App(null, {
            "icon": "fa fa-users",
            "title": "用户管理",
            "src": "__CTX__/static/desktop/tpl/system/index.html",
            "menuId": 3,
            "mode": "default"
        }); 

    //app.show(); 
    //app.hide(); // 最小化
    //app.destroy();    
} );
