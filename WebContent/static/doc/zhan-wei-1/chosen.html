chosen

chosen

1. 使用方式1（格式化原始select表单组件）

硬编码方式，也就是说，<option>全部预先写好，请参考官网 https://harvesthq.github.io/chosen/。

<select name="sysId" 
        data-placeholder="请选择"
        data-pkui-component="chosen">
    <option value="1">选项一</option>
    <option value="2">选项二</option>    
</select>

2. 使用方式2（字典控件）

将其作为字典控件使用。

// desktop/tpl/system/dept/edit.html
<select name="deptLevel"
        data-pkui-component="chosen"
        data-pkui-component-options='{
            "dic":"DIC_DEPT_LEVEL",
            "defaultCode": "{{deptLevel}}",
            "max_shown_results": 5
        }'
></select>

2.1 参数

chosen控件的原始参数都是可以使用的，https://harvesthq.github.io/chosen/options.html。

1）dic （扩展参数）

指定 字典文件的URL，其值可以为：

  • DIC_USER_TYPE （在 proj/static/dic/ 目录下的XML文件 可以直接使用字典名称）
  • http://localhost:8080/proj/static/dic/DIC_USER_TYPE.xml （使用绝对URL）
  • __CTX__/admin/sysDeptDic?deptId=42001&deptLevel=2&releType=4 （请求动态资源，其返回的还是XML文件）

2）defaultCode （扩展参数）

指定默认值。

3）max_shown_results （原始参数）

限制显示的<option>的数量，通常应用于<option>数量过多的情况，比如使用 DIC_CODE.xml 和 DIC_DEPT.xml 。

4）displayInvisibleOptions（扩展参数）

是否显示不可见的选项，即 <row ... DIC_PERM="0"/>（visiable="0"） 的字典条目会显示，默认为 false。

5）disableInvisibleOptions（扩展参数）

是否禁用不可见的选项，默认值 false。配合 displayInvisibleOptions 使用：

  • displayInvisibleOptions = true, disableInvisibleOptions = false：不可见的选项，可显示且可以选择
  • displayInvisibleOptions = true, disableInvisibleOptions = true：不可见的选项，可显示且不可以选择
  • displayInvisibleOptions = false, disableInvisibleOptions = true/false：不可见的选项，不可显示

3. 使用方式3

可以通过 data 属性传入一个数组，避免书写过多的<option>，“方式1”的改进版。

<select name="xx"
        data-pkui-component="chosen"
        data-pkui-component-options='{
            "data": [
                { "value": "=", "text": "=",     "selected": true    },
                { "value": "in", "text": "in" },
                { "value": "like", "text": "like" },
                { "value": "between", "text": "between" }
            ]
        }'
></select>

4. 使用方式4

在“方式3”的基础上，再次增强，通过设置url 属性Ajax请求数据。

// desktop/tpl/system/role/edit.html

/* HTTP Response:
    {
        "success":true,
        "totalRecords":6,
        "data":[
            {"sysId":"subsys_01","sysName":"执法监督综合应用门户"},
            {"sysId":"subsys_02","sysName":"执法问题动态跟踪子系统"},
            {"sysId":"subsys_03","sysName":"如实受立案监督子系统"}
        ]
    }

*/
<select name="sysId" data-placeholder="请选择"
        data-pkui-component="chosen"
        data-pkui-component-options='{
            "url":"{% system.accessSystemlistData %}",
            "selectedValue": "{{sysId}}",
            "optionMapping": {
                "value": "sysId",
                "text": "sysName"
            }
        }'
></select>

/*
    请求 url，返回的数据必须是一个数组（List<Object>或GridResult）。
    数组的每个元素，应该包含 value属性（对应<option>的value） 和 text属性（对应<option>的text），
    如果数组元素没有value属性或text属性，则可以在 optionMapping 中指定。
*/

5. 其他扩展

relateInputName参数

  • 作用：当select上发生change事件时，选中的<option>的text，将注入 <input type="..." name=“relateInputName”>
  • 示例：参考 desktop/tpl/system/role/edit.html
