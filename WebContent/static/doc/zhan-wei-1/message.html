message

message（我的消息）

1. 介绍

在版本1.1.0中，将“我的消息”组件化（模块化），提供如下功能：

  • 自定义消息类别
  • 阅读消息功能，并发送请求标记“已读”
  • 操作反馈：操作行为的信息打印
  • 模板可通过参数指定

2. 使用

2.1 示例

查看：https://github.com/forwardNow/pkui/blob/v1.1.0/app/desktop/index.html

<div 
     data-pkui-component="message"
     data-pkui-component-options='{
        // 请求URL，判断是否有消息，用于在目标元素上添加“有消息”的标志。可不指定。
        // 服务端返回值：JsonResult
        "hasSysMessageURL": "__CTX__/common/hasSysMessage",

        // 请求URL，将指定的消息标志为已被阅读
        // 请求参数名称：msgId
        // 服务端返回值：JsonResult
        "sysMessageMarkCheckedURL": "__CTX__/common/sysMessageMarkChecked",

        /*
         消息类别
            isMarkedMsg：是否为已读消息。该类别的消息是否都是“已读”状态
            text：页签的名称
            icon：页签的图标CSS类，使用fontawesome
            url：请求一页消息，服务器返回值 GridResult
         */   
        "tabList": [
            { "isMarkedMsg": true, "text": "未读", "icon": "fa fa-envelope-o", "url": "__CTX__/common/getUnreadSysMessage?type=all"},
            { "isMarkedMsg": false, "text": "私信", "icon": "fa fa-commenting-o", "url": "__CTX__/common/getUnreadSysMessage?type=private"},
            { "isMarkedMsg": false, "text": "公告", "icon": "fa fa-file-text-o", "url": "__CTX__/common/getUnreadSysMessage?type=public"}
        ],

        // 面板底部 - “更多” ： app参数：以app的方式打开，指定menuId和打开模式mode。
        "moreAppOptions": {
            "menuId": 450,
            "icon": "fa fa-envelope-o",
            "title": "消息管理",
            "mode": "default"
        }
     }'><i class="fa fa-comment-o"></i> 消息 <span class="caret"></span>
</div>

     消息类别

        text：页签的名称

    tabList: null,

    // 请求参数名称：msgId

    placement: "auto",

    // 面板标题
    panelTitle: "我的消息",

        +             '{{if hasLoadInfo}}'

        +                       '<div class="pkui-message-history-list">'

        +                       '</div>'
        +                   '</a>'
