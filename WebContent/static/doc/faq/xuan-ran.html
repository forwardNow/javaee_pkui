2. 组件渲染时机

渲染时机，也就是说在什么时候（对标记为组件的HTML元素）进行渲染。

2.1 DOM元素载入完毕时

对于传统的web应用来说，基本都是表单提交然后重新载入新网页的模式。

因此，在文档DOM树构建完毕后即可进行渲染，通常的做法是：

jQuery( document ).ready( function ( $ ) {
    // 渲染数据表格
    $( "#datagrid" ).bootgrid( { ... } );
    // 渲染菜单树
    $( "#menuTree" ).jstree( { ... } );
    ...
} );

好处：

  • 简单

不足：

  • 对于动态插入的元素无能为力

2.2 用户触发某事件时

当用户对标记为组件的HTML元素进行某个操作（click、keypress）后，进行渲染。

比如，Bootstrap: dropdown.js v3.3.6（bootstrap的下拉菜单插件）：

$(document)
  .on('click.bs.dropdown.data-api', '[data-toggle="dropdown"]', Dropdown.prototype.toggle)
  .on('keydown.bs.dropdown.data-api', '[data-toggle="dropdown"]', Dropdown.prototype.keydown)

当点击元素（[data-toggle="dropdown"]）后，才对其进行渲染。

好处：

  • 对动态添加的元素也有作用
  • 类似于“懒加载”式的“懒渲染”，减少页面载入完毕后渲染的负担

不足：

  • 依赖用户的事件驱动

3. PKUI组件的渲染

PKUI的渲染方法为：

window.PKUI.render();

PKUI.render() 会检索页面所有未被渲染的PKUI组件（没有isrendered属性的元素），并对其进行渲染。

为了降低开发难度，PKUI.render() 会在某些情况下自动执行：

  • DOM树构建完毕，执行一次渲染：jQuery( document ).ready( PKUI.render );
  • 调用$jq.html()方法之后执行一次渲染：AOP.after( $.prototype, "html", PKUI.render )

好处：

  • 对于通过 $jq.html() 动态添加的内容有作用
  • 针对传统的web应用有作用

不足：

  • 对于单页面复杂应用，PKUI.render() 会调用得比较频繁
