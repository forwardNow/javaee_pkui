表单重复提交问题

表单重复提交问题

会产生重复提交数据的情况，可以以数据提交方式分为两种：form提交、ajax提交。

1. form提交数据

如下这类情形：

// 插入
<form action="/proj/admin/saveSysDept">
    部门名称：<input type="text" name="deptName">
    ......
    <input type="submit" value="提交">
</form>

// 修改
<form action="/proj/admin/saveSysDept">
    <input type="hidden" name="deptId" value="dept_001">
    部门名称：<input type="text" name="deptName">
    ......
    <input type="submit" value="提交">
</form>

这种方式，通过点击“提交”按钮或执行$form.submit() 来提交数据。

特点：会刷新（载入新的文档）。

1.1 描述

表单提交后，服务器端会进行相应的处理，最后决定是转发（forward）还是重定向（redirect）：

  • 重定向：会产生两个HTTP请求，处理表单的URL1（<form>的action指定）和告知浏览器载入的新文档的URL2（在针对URL1的HTTP响应中读取）。也就是说，在服务器处理完毕表单数据后，不直接返回新的文档，而是返回一个URL，让浏览器根据返回的URL请求新的文档。也就是说，你找服务器“借钱”，服务器不直接借钱你，而是告诉你张三的地址，让你找张三借钱，最后你找张三借到了钱。
  • 转发：只会有一个HTTP请求（处理表单的URL1），一步处理到位，即在服务器处理完毕表单数据后，直接返回新的文档给浏览器。也就是说，你找服务器“借钱”，服务器找张三借到钱后，再把钱给你。

发生重复提交的情况，以用户可以操作（非恶意的操作）的时机来分：

  1. 表单填写完毕（输入域有效性检验通过），点击“提交”按钮后，由于网络慢或浏览器处理慢，导致页面没反应
    • [情况1]用户再次（或多次）点击“提交”按钮后，导致重复提交数据
  2. 服务器端处理完毕，浏览器呈现新的文档，解析此文档的过程中，浏览器是一片空白，可能会导致用户 点击刷新按钮或在地址栏敲回车按钮 来重新载入文档，浏览器再次发送了一个请求（跟浏览器地址栏一样的URL）
    • 如果是重定向，重复的请求只是请求新的文档，而不会再次处理表单数据，所以不会发生重复提交
    • [情况2]如果是转发，重复的请求 会把处理表单数据和返回新的文档的过程 再走一遍，因此导致了重复提交
  3. 用户点击浏览器“后退”按钮，再次进入表单录入界面。当然此种情况不在讨论的范围，这里只讨论无意的重复HTTP请求。

1.2 解决方案

[情况1]多次点击“提交”按钮导致的重复提交。在第一次点击“提交”按钮后，禁用“提交”按钮，即可解决。

[情况2]刷新导致的重复提交。在服务器端使用token。

2. Ajax提交数据

Ajax提交数据，不会刷新页面，而且大多数情况下都是异步的请求。

由于不会刷新页面，所以不会有[情况2]的发生。

2.1 描述

[情况3]由于ajax与服务器交互的过程及结果是“无声无息”的，如果页面做不到及时且有效的反馈，也会导致用户多次点击“提交”按钮。

2.2 解决方案

[情况3]多次点击“提交”按钮导致的重复提交。可以采用下列步骤以解决重复提交：

  1. 表单填写完毕
  2. 点击“提交”按钮后，禁用“提交”按钮
  3. 服务器正在处理请求以及返回结果的过程，显示处理进度（进度条或loading状态）
  4. 浏览器接收到HTTP响应后，根据结果对用户进行反馈
    • 处理成功，则提示处理结果后，直接隐藏（或关闭）表单填写界面
    • 处理失败，则提示失败后，取消按钮的禁用状态和loading状态，继续留在表单界面，让用户可以修正数据后再次提交

3. 在PKUI中具体实现

由于 desktop 是类桌面的模式，所以，全部采用的是将表单数据序列化后以Ajax的方式提交。

其提交用户数据的模式为两种：

  • [模式1]点击“提交”按钮后，提交少量数据，比如 删除按钮
  • [模式2]点击“新增”按钮后，弹出一个表单输入界面，在表单数据界面上进行数据的录入和提交

[模式1]：

/* 参考：desktop/tpl/system/dic/list/list.html */

<button type="button" class="btn btn-primary-2"
        onclick="doCreateDicListDic(this)"
        title="1、将指定字典（勾选的记录）生成字典文件；2、（不勾选任何记录时）将所有字典生成字典文件"
        ><i class="fa fa-book"></i> 生成字典</button>

function doCreateDicListDic( button ) {
    var
        $button = jQuery( button ),
        $table = jQuery( "#sysDicListDatagrid" ),
        seletedRowIds = $table.bootgrid( "getSelectedRows" ),
        data = ""
        ;
    // 如果处于 isloading 状态，则退出
    if ( $button.attr( "disabled" ) ) {
        return;
    }

    // 打开 loading
    $button.isLoading( { position: "insideButton" } );

    // 将 [ 1, 2, 3 ] 转换为 "dicName=1&dicName=2&dicName=3"
    $.each( seletedRowIds, function ( index, dicName ) {
        if ( index > 0 ) {
            data += "&";
        }
        data += "dicName=" + dicName;
    } );

    jQuery.ajax( {
        url: "{% system.dic.list.list.sysDicCreateXml %}",
        data: data
    } ).done( function ( jsonResult ) {
        if ( jsonResult && jsonResult.success ) {
            var dicName
                ;
            if ( data ) {
                dicName = data.split( "&" );
            }
            // 清空字典缓存
            window.PKUI.component.DataSource.clearCache( dicName );
            layer.msg( jsonResult.message || "字典生成成功！", { icon: 1 } );
        }
        else {
            layer.alert( ( jsonResult && jsonResult.message ) || "字典生成失败！", { icon: 2 } );
        }
    } ).fail( function () {
        // 提示网络错误
        layer.alert( '网络错误！', { icon: 0 } );
    } ).always( function () {
        // 关闭 loading
        $button.isLoading( "hide" );
    } );
}
