datagrid

数据表格（bootgrid）

1. 介绍

  • PKUI采用bootgrid作为其数据表格（datagrid）。
  • 官方文档：http://www.jquery-bootgrid.com/Documentation

2. 使用

<table id="sysDeptDatagrid"
       // 标志使用 datagrid 
       data-pkui-component="datagrid"

       data-pkui-component-options='{
            "selection": true,

2.1 一般设置（针对整体）

参考：http://www.jquery-bootgrid.com/Documentation#table

设置位置：data-pkui-component-options='{ ... }'

常用参数说明：

navigation

  • 作用：数据表格的结构设置。也就是说决定 “header”（隐藏的表格设置）和“footer”（分页条）的显示。
  • 值：Number。0 - 都不显示；1 - 只显示“header”；2 - 只显示“footer”；3 - 全部显示（默认）。

selection

  • 作用：让该行可被选中，也就是说添加一列复选框，让该列可以被勾选，只能单选。
  • 值：Boolean。false（默认）。

multiSelect

  • 作用：配合 selection 使用，开启多选。
  • 值：Boolean。false（默认）。

padding

  • 作用：控制分页条的条目个数。
  • 值：Number。4（默认）。
  • 示例：显示第6页时的分页条，第2页 | 第3页 | 第4页 | 第5页 | 第6页 | 第7页 | 第8页 | 第9页 | 第10页

2.2 列设置

参考：http://www.jquery-bootgrid.com/Documentation#column

设置位置：<thead> <tr> <th 参数名="参数值"></th> <tr></thead>

常用参数说明：

data-column-id

  • 作用：对应bean的属性名
  • 示例：<th data-column-id="deptName" ...></th>

data-identifier

  • 作用：记录的唯一性标记。也就是说，data-column-id 指定的bean属性是ID属性。
  • 值：Boolean。false（默认）。只能对一个<th>设置为true。
  • 示例：<th data-column-id="deptId" data-identifier="true" ...>

data-visible

  • 作用：该列是否显示。通常让主键列不显示。
  • 值：Boolean。true（默认）。
  • 示例：<th data-column-id="deptId" data-identifier="true" data-visible="false">

data-sortable

  • 作用：该列是否可排序。点击该列的标题时，会拼接“order by”子句发送服务器。
  • 值：Boolean。true（默认）
  • 示例：<th ... data-sortable="true" ...>部门级别</th>

data-order

  • 作用：指定初始化时的列顺序。sortable 仅仅指定该列可排序，你需要点击列标题才能进行排序；而order 则在可排序的基础上，预定义排序，也就是说指定该列的默认排序。
  • 值：String。""（默认）；"asc"；"desc"。
  • 示例：<th ... data-sortable="true" data-order="asc" ...>部门级别</th>

data-width

  • 作用：指定列的宽度。不要每列都设置宽度，不然多出的宽度会平均分配到每一列，不够的宽度会平均扣除。
  • 值：百分比 或 Number。指定为Number时，其单位为“px”。
  • 示例：<th data-width="30%">部门名称</th>

data-formatter

  • 作用：内容格式化器。也就是说，对原始的属性值进行格式化。
  • 值：全局函数的函数名 或者 内置的格式化器的名称。
  • 内置格式化器
    • <th data-formatter="dateFormatter" ...> 格式化日期，参考FAQ第10条
    • <th data-formatter="datetimeFormatter" ...> 格式化日期时间，参考FAQ第10条
    • statusFormatter：标志状态（正常/停用）。"1" - 正常，"0" - 停用，其他值 - ""（空串）。
    • yesOrNoFormatter：标志状态（是/否）。"1" - 是，"0" - 否，其他值 - ""（空串）。
  • 全局格式化器：<th data-formatter="sysDeptOpe">操作</th>

data-dic （扩展属性）

  • 作用：使用指定的字典翻译该属性值。
  • 值：String。字典名称 或 绝对路径。
  • 示例：<th data-dic="DIC_DEPT_LEVEL" ...>

data-has-dropdown-menu （扩展属性）

  • 作用：设置该列的单元格的“overflow:auto;”。也就是说溢出的部分仍旧显示，不会出现下拉菜单被遮罩的问题。通常使用在操作列（最后一列）。
  • 值：Boolean。false（默认）。

2.3 API

2.3.1 常用

  • getSelectedRows ：
    • 参数，无；
    • 作用，获取选中记录的ID列表；
    • 返回值， [ "id-1", "id-2" ]

2.3.2 扩展

  • getSelectedRowsDataList ：
    • 参数，无；
    • 作用，获取选中记录的data列表；
    • 返回值， [ { id:1, name: "x" }, { id:2, name: "y" } ]
  • deleteRow ：
    • 参数， ID列表（[ 1,2,3 ] ）；
    • 作用，标志那行被删除；
    • 返回值，无。

3. 扩展

3.1 列调整

默认使用了 colResizable.js（pkui/lib/jquery/plugin/colResizable） ，可通过拖拽列来调整列宽。

3.2 操作列的自适应（未完成）

由于，数据表格的单元格内容是 单行显示且超过部分显示为“...”，

所以，当操作列的功能链接（功能按钮）过多（3个以上）时，会出现显示不全的问题。

因此，当数据表格初始化完毕时，应将多余的功能链接以“更多”的方式显示，

即，点击“更多”会弹出一个下拉菜单 以 显示额外的功能链接。

3.3 关联表单

3.3.1 介绍

作用：附加额外的查询参数

设置位置：<table ... data-query-form-selector="#sysDeptQueryForm" ...>

示例：

<form id="sysMessageQueryForm">
    <div class="input-group">
        <span class="input-group-addon">本人角色</span>
        <select name="isSend" id="sysMessage-isSend" data-not-in-criteria="true"
                data-pkui-component="chosen">
            <option value="false">接收者</option>
            <option value="true">创建者</option>
        </select>
    </div>
    <div class="input-group">

    <button class="btn btn-default-2" type="button"
            onclick='jQuery( "#sysMessageDatagrid" ).bootgrid( "reload" );'><i class="fa fa-refresh"></i></button>
</form>

3.3.2 输入域说明

交互的数据格式，请参考 分页问题。

data-not-in-criteria=true

  • 作用：标志该输入域不会添加到 oredCriteria，但依旧会添加的请求中
  • 示例：
  • isSend:true  // 此属性会添加到查询字符串中，但不会添加到 “txtQuery”

start:0
limit:10
pageSize:10
totalRecords:0
txtQuery:{"oredCriteria":[[]],"orderByClause":"","pager":{"start":0,"limit":10,"pageSize":10,"totalRecords":0}}
pageIndex:1
size:10
3.3.2.1 操作符（data-operator）
  • =：默认
  • !=：不等于
  • <>：不等于
  • >：大于
  • >=：大于等于
  • <：小于
  • <=：小于等于
  • LIKE：模糊查询
  • IN：集合操作
  • NOT IN：集合操作
  • NULL：为空
  • NOT NULL：不为空
3.3.2.2 值的类型（data-datatype）

data-datatype="1"

  • 长整形

data-datatype="3"

  • 日期类型
  • 去掉分隔符后的值得满足格式“yyyyMMdd”。
  • 示例：2017-09-14 或 2017/09/14 或 20170914

data-datatype="4"

  • 日期时间类型（时间戳类型）
  • 去掉分隔符后的值得满足格式“yyyyMMddHHmmss”。
  • 示例：2017-09-14 16:43:22 或 20170914164322

data-datatype="7"

  • 浮点型

data-datatype=“其他值”（或不填）

  • 原始字符串
3.3.2.3 示例

精确查询

<input type="text" name="deptName" data-operator="=" value="武汉">
<input type="text" name="deptName" value="武汉">
转换后的SQL语句：and DEPT_NAME = '武汉'

模糊查询

<input type="text" name="deptName" data-operator="like" value="武汉">
转换后的SQL语句：and DEPT_NAME like '%武汉%'

小于

<input type="text" name="deptName" data-operator="&lt;" value="武汉">

3.4 删除功能

3.4.1 说明

由于删除功能不是很复杂，且很通用，因此对其进行封装成PKUI组件。

参考：pkui/component/logic/datagridExt/1.0.0/datagrid-delete.js

3.4.2 使用

<button type="button" class="btn btn-danger-2"
        data-pkui-component="datagrid-delete"
        data-pkui-component-options='{
            // 绑定bootgrid
            "datagridSelector": "#sysMessageDatagrid",
            // 要删除的记录的主键属性名

            "url": "{% common.msg.list.sysMessageDelete %}",
            // 额外的请求参数
            "extraData": {
                // 请求参数名：输入域选择器
                "isSend": "#sysMessage-isSend"

        }'
        title="删除勾选的记录（可多选）"
        ><i class="fa fa-trash-o"></i> 删除</button>
