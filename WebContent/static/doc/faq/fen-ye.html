1. 数据交换格式

1.1 前端（request）

1）新的请求格式

{
    pageIndex: 数字值, // 页索引。从1开始。（请求第几页）
    size: 数字值,      // 页的行数。（每页的最大记录数）

    xxx: yyy,               // 扩展参数（额外的参数）   
    xxx: yyy,
    ......
}

// 获取总数的请求格式（Http Request）
{
    xxx: yyy,               // 扩展参数（额外的参数）   
    xxx: yyy,
    ......
}

// 获取总数的响应格式（Http Response）
xxx    // 数字 或 数字字符串

2）老的请求格式

{
    start: 数字值,
    limit: 数字值,
    pageSize: 数字值,
    totalRecords: 数字值,
    txtQuery: {
        "oredCriteria": [
            [
                {"property":"loginName","operator":"like","value":"%admin%","datatype":"0"}
            ]
        ],
        "orderByClause": "LOG_TIME DESC",
        "pager": {
            "start":0,
            "limit":10,
            "pageSize":10,
            "totalRecords":1318
        }
    }
}

3）现行的请求格式（兼容以上，合并）

{
    pageIndex: 数字值, // 页索引。从1开始。（请求第几页）
    size: 数字值,      // 页的行数。（每页的最大记录数）

    xxx: yyy,
    ......


    start: 数字值,
    limit: 数字值,
    pageSize: 数字值,
    totalRecords: 数字值,
    txtQuery: ...,
}

2. 分页

2.1 有总数的分页（请求第1页时计算总数，请求其他页时则不计算总数）

1）新后台框架（使用新的请求格式）

  • 标志需要总数：页面设置“data-total-records-url”属性
  • 请求总数的时机：在处理完服务器端返回的数据（第1页）时，发送同步Ajax请求总数
// 页面
<table id="sysLogDatagrid"
       data-pkui-component="datagrid"
       data-pkui-component-options='{
            "selection": true,
            "multiSelect": true
       }'
       data-url="__CTX__/admin/sysLogListData_new_getPage"

       data-total-records-url="__CTX__/admin/sysLogListData_new_getCount"

       data-query-form-selector="#sysLogQueryForm"
       >

2）老后台框架（使用老的请求格式）

  • 标志需要总数：服务器端在返回第1页的数据中，totalRecords != 0 && data != null
@RequestMapping( "/admin/sysLogListData_old_totalRecords" )
@ResponseBody
public GridResult sysLogListData_old_totalRecords(String txtQuery) {
    try {
        int count = 0;
        Criteria<?> criteria = WebUtils.toCriteria(txtQuery);
        List<SysLog> list = sysLogService.getListByCriteria(criteria);
        count = criteria.getPager().getTotalRecords();
        if ( count == 0 ) {
            count = sysLogService.getCountByCriteria(criteria);
        }
        return new GridResult(true, list, count);
    } catch (Exception e) {
        logger.error("查询列表数据出错", e);
        return new GridResult(false, null);
    }
}

2.2 无总数的分页（无限下一页模式）

1）新后台框架（使用新的请求格式）

标志不需要总数：不设置“data-total-records-url”属性

2）老后台框架（使用老的请求格式）

标志不需要总数：无论请求哪一页，服务器端返回的数据中， totalRecords == 0 && data != null；

@RequestMapping( "/admin/sysLogListData_old_noTotalRecords" )
@ResponseBody
public GridResult sysLogListData_old_noTotalRecords(String txtQuery) {
    try {
        Criteria<?> criteria = WebUtils.toCriteria(txtQuery);
        List<SysLog> list = sysLogService.getListByCriteria(criteria);
        int count = 0;
        return new GridResult(true, list, count);
    } catch (Exception e) {
        logger.error("查询列表数据出错", e);
        return new GridResult(false, null);
    }
}
