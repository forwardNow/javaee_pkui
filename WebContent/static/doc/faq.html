FAQ（Frequently Asked Questions）

1、如何获取PKUI最新版本？

可以从GitHub获取下载最新版本：https://github.com/forwardNow/pkui。

pkui（/）
|-- app
|   |-- desktop    // => /dist/desktop
|
|-- src    // => /dist/pkui
|
|-- dist
    |-- pkui
    |-- desktop

2、如何使用PKUI，有没有Demo之类的？

可以访问已经使用PKUI的在线项目：http://www.20ui.cn/ 。

可以从GitHub下载该项目发布的最新版本的源码：https://github.com/forwardNow/javaee_pkui/releases 。

pkui（/）
|-- WebContent
    |-- static
        |-- pkui
        |-- desktop

3、如何在项目中使用PKUI？

先 下载在线项目javaee_pkui（参阅问题2），

然后 将javaee_pkui项目的static目录下的 pkui、desktop 拷贝到 你的项目的static目录下，

最后引入核心JS文件、CSS文件即可，如下

// 在HTML中引入，参考 /pkui/WebContent/static/desktop/index.html
<link rel="stylesheet" href="css/style.css">
<script data-main="./js/main.js" data-timestamp="2017-07-06" src="../pkui/pkui.js"></script>

// 在JSP中引入，参考 /pkui/WebContent/WEB-INF/views/login.jsp
<link rel="stylesheet" href="${ ctx }/static/desktop/css/style.css">
<script data-main="${ ctx }/static/js/main.js" data-timestamp="2017-03-22" 
        src="${ ctx }/static/pkui/pkui.js"></script>

4、在HTML（非JSP）中如何使用绝对路径？

以前采用JSP作为模板引擎，可以使用 ${ctx}/admin/saveSysUser 这种绝对路径；

但现在采用前端（web端）模板引擎，在HTML页面可使用绝对路径 __CTX__/admin/saveSysUser 。

在JSP被翻译为HTML的过程中，${ctx} 会被替换为 http://localhost:8080/pkui ；

在使用 $jquery.html( HTML )、$.ajax( OPTIONS )、<div data-pkui-component-options='VALUE'>时，

HTML、OPTIONS的url、VALUE 中的 __CTX__ 会被替换为 http://localhost:8080/pkui 。

5、__CTX__ 如何得到的？

project
|-- static
    |-- pkui
        |-- pkui.js

通过 <script ... src="../pkui/pkui.js"></script> 引入 pkui.js 后，

pkui.js 在执行时，获取该 <script> 的绝对路径 http://localhost:8080/pkui/static/pkui/pkui.js ，

解析此绝对路径，定位 /static （以此为锚点），则 __CTX__ = http://localhost:8080/pkui 。

可查看解析出来的其他路径：

// __CTX__
window.PKUI.ctxPath    //=> "http://localhost:8080/pkui"

// Sea.js 的基础路径
window.PKUI.basePath   //=> "http://localhost:8080/pkui/static/pkui"

7、如何配置HTML页面中的URL占位符？

请参阅 systemUrl.js 文件内容及其注释：

/
|-- static
    |-- desktop
        |-- js
            |-- config
                |-- systemUrl.js

在 systemUrl.js 文件中定义：

{
    "system.dept.list.sysDeptListData": "__CTX__/admin/sysDeptListData",
    ......
}

使用systemUrl.js文件中定义的占位符：

// 1. $jquery.html( HTML ), 在 “HTML” 中使用
// 如 desktop/tpl/system/dept/list.html
<table id="sysDeptDatagrid"
       data-pkui-component="datagrid"
       data-pkui-component-options='{
            "selection": true,
            "multiSelect": true
       }'
       data-url="{% system.dept.list.sysDeptListData %}" ......

// 2. <div data-pkui-component-options='VALUE' ...> 在“VALUE”中使用
// 如 desktop/tpl/system/dept/list.html
<button type="button" class="btn btn-danger-2"
            data-pkui-component="datagrid-delete"
            data-pkui-component-options='{
                "datagridSelector": "#sysDeptDatagrid",
                "PKName": "deptId",
                "url": "{% system.dept.list.sysDeptDelete %}"
            }'
            title="删除勾选的记录（可多选）"
            ><i class="fa fa-trash-o"></i> 删除</button>

// 3. $.ajax( OPTIONS )，在“OPTIONS的url”中使用

9、当pkui目录下没有满足项目要求的组件（插件），如何使用自己从网上下载下来的插件呢？

proj
|-- static
    |-- desktop
    |   |-- css
    |       |-- style.css
    |-- pkui
    |   |-- pkui.js
    |   |-- lib    
    |       |-- jquery
    |       |   |-- 1.11.3.x
    |       |       |-- jquery.js
    |-- js
        |-- yourPlugin.js  // 你要引入的插件

传统的引入插件的方式

<script src="${ctx}/static/pkui/lib/jquery/1.11.3.x/jquery.js"></script>
<script src="${ctx}/static/js/yourPlugin.js"></script>

现在引入插件的方式

<link rel="stylesheet" href="${ ctx }/static/desktop/css/style.css">
<script data-main="xx" data-timestamp="2017-03-22" src="${ ctx }/static/pkui/pkui.js"></script>
<script>
    seajs.use( [ "jquery", window.PKUI.ctxPath + "/static/js/yourPlugin"  ], function( $, yourPlugin ) {
        // do something
    } );
</script>

传统编写插件的方式

+function($, window){
    ......
    window.yourPlugin = YourPlugin;
}( jQuery, this );

现在编写插件的方式

define( function( require ) {
    var $ = require( "jquery" );
    ......
    return YourPlugin;
} );

10、菜单的图标、URL怎么设置？

快捷方式的图标主要通过手动指定而来，而菜单树上的菜单项的URL和icon取自哪里呢？

一般情况下，菜单项的

  • 图标取自 SYS_MENU.ICON
  • URL取自 SYS_MENU.MENU_URL

但是，如果占用 SYS_MENU.ICON 、SYS_MENU.MENU_URL 这两个字段，会导致一些问题，比如

  • 新系统（PKUI）出问题时，无法回退到老系统
  • 无法使用老系统丰富的图标

所以，使用了 SYS_MENU.RESERVE1（图标）、SYS_MENU.RESERVE2（URL） 这两个字段

  • SYS_MENU.RESERVE1（图标） ：当此字段的值为字体图标的样式类时，则使用字体图标，否则使用SYS_MENU.ICON
  • SYS_MENU.RESERVE2（URL） ：当此字段的值为URL时，则使用此URL，否则使用 SYS_MENU.MENU_URL

弊端，新的解决方案带来的问题

  • SYS_MENU.RESERVE1、SYS_MENU.RESERVE2的优先级高于 SYS_MENU.ICON 、SYS_MENU.MENU_URL
  • 菜单管理修改的是SYS_MENU.ICON 、SYS_MENU.MENU_URL而非SYS_MENU.RESERVE1、SYS_MENU.RESERVE2，当保留字段有相应的值时，IOCN和MENU_URL字段 的修改无效的。

因此，新系统（PKUI）采用 SYS_MENU.RESERVE1、SYS_MENU.RESERVE2 存储图标和URL。

11、数据表格（bootgrid/datagrid）如何处理记录中的日期属性？

数据表格可以处理的日期属性值如下：

// 日期形式1：毫秒数（距离1970-01-01的毫秒数）
[
    { "deptId": 1, "addTime": 1500428308802, ... }, ...
]

// 日期形式2：YYYY-MM-DD 或 YYYY-MM-DD HH:mm:ss  形式的字符串
[
    { "deptId": 1, "addTime": "2017-07-19", ... }, ...
    { "deptId": 1, "addTime": "2017-07-19 09:46:03", ... }, ...
]

// 日期形式3：YYYY/MM/DD 或 YYYY/MM/DD HH:mm:ss 形式的字符串
[
    { "deptId": 1, "addTime": "2017/07/19", ... }, ...
    { "deptId": 1, "addTime": "2017/07/19 09:46:03", ... }, ...
]

// 日期形式4：YYYYMMDD 或 YYYYMMDDHHmmss 形式的字符串
[
    { "deptId": 1, "addTime": "20170719", ... }, ...
    { "deptId": 1, "addTime": "20170719094603", ... }, ...
]

数据表格 将 日期格式的数据 格式化成什么格式的日期：

/* 参见：desktop/js/config/globalSetting.js */

    // 数据表格使用 dateFormatter 时，日期格式化后的日期
    PKUI.datagridDateFormat = "YYYY-MM-DD";

    // 数据表格使用 datetimeFormatter 时，日期时间格式化后的日期时间
    PKUI.datagridDatetimeFormat = "YYYY-MM-DD HH:mm:ss";

使用格式器：

// 日期（如 2017-07-19）
<th data-column-id="addTime" data-formatter="dateFormatter" ...>录入时间</th>
// 时间戳（如 2017-07-19 09:46:03）
<th data-column-id="addTime" data-formatter="datetimeFormatter" ...>录入时间</th>
